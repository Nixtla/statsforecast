## <kbd>class</kbd> `AutoARIMA`
An AutoARIMA estimator. 

Returns best ARIMA model according to either AIC, AICc or BIC value. The function conducts a search over possible model within the order constraints provided. 



**Args:**
 
 - <b>`d`</b> (int, optional):  Order of first-differencing.  If missing, will choose a value based on `test`. Defaults to None. 
 - <b>`D`</b> (int, optional):  Order of seasonal-differencing.  If missing, will choose a value based on `season_test`. Defaults to None. 
 - <b>`max_p`</b> (int):  Maximum value of p. Defaults to 5. 
 - <b>`max_q`</b> (int):  Maximum value of q. Defaults to 5. 
 - <b>`max_P`</b> (int):  Maximum value of P. Defaults to 2. 
 - <b>`max_Q`</b> (int):  Maximum value of Q. Defaults to 2. 
 - <b>`max_order`</b> (int):  Maximum value of p+q+P+Q if model selection is not stepwise. Defaults to 5. 
 - <b>`max_d`</b> (int):  Maximum number of non-seasonal differences. Defaults to 2. 
 - <b>`max_D`</b> (int):  Maximum number of seasonal differences. Defaults to 1. 
 - <b>`start_p`</b> (int):  Starting value of p in stepwise procedure. Defaults to 2. 
 - <b>`start_q`</b> (int):  Starting value of q in stepwise procedure. Defaults to 2. 
 - <b>`start_P`</b> (int):  Starting value of P in stepwise procedure. Defaults to 1. 
 - <b>`start_Q`</b> (int):  Starting value of Q in stepwise procedure. Defaults to 1. 
 - <b>`stationary`</b> (bool):  If True, restricts search to stationary models. Defaults to False. 
 - <b>`seasonal`</b> (bool):  If False, restricts search to non-seasonal models. Defaults to True. 
 - <b>`ic`</b> (str):  Information criterion to be used in model selection. Defaults to 'aicc'. 
 - <b>`stepwise`</b> (bool):  If True, will do stepwise selection (faster).  Otherwise, it searches over all models.  Non-stepwise selection can be very slow,  especially for seasonal models. Defaults to True. 
 - <b>`nmodels`</b> (int):  Maximum number of models considered in the stepwise search. Defaults to 94. 
 - <b>`trace`</b> (bool):  If True, the list of ARIMA models considered will be reported. Defaults to False. 
 - <b>`approximation`</b> (bool, optional):  If True, estimation is via conditional sums of squares  and the information criteria used for model  selection are approximated.  The final model is still computed using  maximum likelihood estimation.  Approximation should be used for long time series  or a high seasonal period to avoid excessive computation times. Defaults to None. 
 - <b>`method`</b> (str, optional):  fitting method: maximum likelihood or minimize conditional  sum-of-squares.  The default (unless there are missing values)  is to use conditional-sum-of-squares to find starting values,  then maximum likelihood. Can be abbreviated. Defaults to None. 
 - <b>`truncate`</b> (bool, optional):  An integer value indicating how many observations  to use in model selection.  The last truncate values of the series are  used to select a model when truncate is not None  and approximation=True.  All observations are used if either truncate=None  or approximation=False. Defaults to None. 
 - <b>`test`</b> (str):  Type of unit root test to use. See ndiffs for details. Defaults to 'kpss'. 
 - <b>`test_kwargs`</b> (str, optional):  Additional arguments to be passed to the unit root test. Defaults to None. 
 - <b>`seasonal_test`</b> (str):  This determines which method is used to select the number  of seasonal differences.  The default method is to use a measure of seasonal  strength computed from an STL decomposition.  Other possibilities involve seasonal unit root tests. Defaults to 'seas'. 
 - <b>`seasonal_test_kwargs`</b> (dict, optional):  Additional arguments to be passed to the seasonal  unit root test. See nsdiffs for details. Defaults to None. 
 - <b>`allowdrift`</b> (bool):  If True, models with drift terms are considered. Defaults to True. 
 - <b>`allowmean`</b> (bool):  If True, models with a non-zero mean are considered. Defaults to True. 
 - <b>`blambda`</b> (float, optional):  Box-Cox transformation parameter.  If lambda="auto", then a transformation is automatically  selected using BoxCox.lambda.  The transformation is ignored if None.  Otherwise, data transformed before model is estimated. Defaults to None. 
 - <b>`biasadj`</b> (bool):  Use adjusted back-transformed mean for Box-Cox transformations.  If transformed data is used to produce forecasts and fitted values,  a regular back transformation will result in median forecasts.  If biasadj is True, an adjustment will be made to produce  mean forecasts and fitted values. Defaults to False. 
 - <b>`period`</b> (int):  Number of observations per unit of time.  For example 24 for Hourly data. Defaults to 1. 



**Notes:**

> * This implementation is a mirror of Hyndman's forecast::auto.arima. 
>References: [1] https://github.com/robjhyndman/forecast

### <kbd>method</kbd> `fit`

```python
fit(y: ndarray, X: Optional[ndarray] = None)
```

Fit the AutoARIMA estimator. 

Fit an AutoARIMA to a time series (numpy array) `y` and optionally exogenous variables (numpy array) `X`. 



**Args:**
 
 - <b>`y`</b> (np.ndarray):  One-dimensional numpy array of floats without `np.nan` or `np.inf`  values. 
 - <b>`X`</b> (np.ndarray, optional):  An optional 2-d numpy array of exogenous variables (float).  Defaults to None.

### <kbd>method</kbd> `predict`

```python
predict(
    h: int,
    X: Optional[ndarray] = None,
    level: Optional[int, Tuple[int]] = None
)
```

Forecast future values using a fitted AutoArima. 



**Args:**
 
 - <b>`h`</b> (int):  Number of periods for forecasting. 
 - <b>`X`</b> (np.ndarray, optional):  Future exogenous variables. Defaults to None. 
 - <b>`level`</b> (int or tuple of int, optional):  Confidence level for prediction intervals. Defaults to None. 



**Returns:**
 
 - <b>`pd.DataFrame`</b>:  The array of fitted values.  The confidence intervals for the forecasts are returned if  level is not None.
