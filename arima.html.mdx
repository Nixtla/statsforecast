### `AutoARIMA`

```python
AutoARIMA(
    d=None,
    D=None,
    max_p=5,
    max_q=5,
    max_P=2,
    max_Q=2,
    max_order=5,
    max_d=2,
    max_D=1,
    start_p=2,
    start_q=2,
    start_P=1,
    start_Q=1,
    stationary=False,
    seasonal=True,
    ic="aicc",
    stepwise=True,
    nmodels=94,
    trace=False,
    approximation=None,
    method=None,
    truncate=None,
    test="kpss",
    test_kwargs=None,
    seasonal_test="seas",
    seasonal_test_kwargs=None,
    allowdrift=True,
    allowmean=True,
    blambda=None,
    biasadj=False,
    period=1,
)
```

An AutoARIMA estimator.

Returns best ARIMA model according to either AIC, AICc or BIC value.
The function conducts a search over possible model within the order constraints provided.

**Parameters:**

Name | Type | Description | Default
---- | ---- | ----------- | -------
`d` | <code>[int](#int)</code> | Order of first-differencing.
If missing, will choose a value based on `test`. Defaults to None. | <code>None</code>
`D` | <code>[int](#int)</code> | Order of seasonal-differencing.
If missing, will choose a value based on `season_test`. Defaults to None. | <code>None</code>
`max_p` | <code>[int](#int)</code> | Maximum value of p. Defaults to 5. | <code>5</code>
`max_q` | <code>[int](#int)</code> | Maximum value of q. Defaults to 5. | <code>5</code>
`max_P` | <code>[int](#int)</code> | Maximum value of P. Defaults to 2. | <code>2</code>
`max_Q` | <code>[int](#int)</code> | Maximum value of Q. Defaults to 2. | <code>2</code>
`max_order` | <code>[int](#int)</code> | Maximum value of p+q+P+Q if model selection is not stepwise. Defaults to 5. | <code>5</code>
`max_d` | <code>[int](#int)</code> | Maximum number of non-seasonal differences. Defaults to 2. | <code>2</code>
`max_D` | <code>[int](#int)</code> | Maximum number of seasonal differences. Defaults to 1. | <code>1</code>
`start_p` | <code>[int](#int)</code> | Starting value of p in stepwise procedure. Defaults to 2. | <code>2</code>
`start_q` | <code>[int](#int)</code> | Starting value of q in stepwise procedure. Defaults to 2. | <code>2</code>
`start_P` | <code>[int](#int)</code> | Starting value of P in stepwise procedure. Defaults to 1. | <code>1</code>
`start_Q` | <code>[int](#int)</code> | Starting value of Q in stepwise procedure. Defaults to 1. | <code>1</code>
`stationary` | <code>[bool](#bool)</code> | If True, restricts search to stationary models. Defaults to False. | <code>False</code>
`seasonal` | <code>[bool](#bool)</code> | If False, restricts search to non-seasonal models. Defaults to True. | <code>True</code>
`ic` | <code>[str](#str)</code> | Information criterion to be used in model selection. Defaults to 'aicc'. | <code>'aicc'</code>
`stepwise` | <code>[bool](#bool)</code> | If True, will do stepwise selection (faster).
Otherwise, it searches over all models.
Non-stepwise selection can be very slow,
especially for seasonal models. Defaults to True. | <code>True</code>
`nmodels` | <code>[int](#int)</code> | Maximum number of models considered in the stepwise search. Defaults to 94. | <code>94</code>
`trace` | <code>[bool](#bool)</code> | If True, the list of ARIMA models considered will be reported. Defaults to False. | <code>False</code>
`approximation` | <code>[bool](#bool)</code> | If True, estimation is via conditional sums of squares
and the information criteria used for model
selection are approximated.
The final model is still computed using
maximum likelihood estimation.
Approximation should be used for long time series
or a high seasonal period to avoid excessive computation times. Defaults to None. | <code>None</code>
`method` | <code>[str](#str)</code> | fitting method: maximum likelihood or minimize conditional
sum-of-squares.
The default (unless there are missing values)
is to use conditional-sum-of-squares to find starting values,
then maximum likelihood. Can be abbreviated. Defaults to None. | <code>None</code>
`truncate` | <code>[bool](#bool)</code> | An integer value indicating how many observations
to use in model selection.
The last truncate values of the series are
used to select a model when truncate is not None
and approximation=True.
All observations are used if either truncate=None
or approximation=False. Defaults to None. | <code>None</code>
`test` | <code>[str](#str)</code> | Type of unit root test to use. See ndiffs for details. Defaults to 'kpss'. | <code>'kpss'</code>
`test_kwargs` | <code>[str](#str)</code> | Additional arguments to be passed to the unit root test. Defaults to None. | <code>None</code>
`seasonal_test` | <code>[str](#str)</code> | This determines which method is used to select the number
of seasonal differences.
The default method is to use a measure of seasonal
strength computed from an STL decomposition.
Other possibilities involve seasonal unit root tests. Defaults to 'seas'. | <code>'seas'</code>
`seasonal_test_kwargs` | <code>[dict](#dict)</code> | Additional arguments to be passed to the seasonal
unit root test. See nsdiffs for details. Defaults to None. | <code>None</code>
`allowdrift` | <code>[bool](#bool)</code> | If True, models with drift terms are considered. Defaults to True. | <code>True</code>
`allowmean` | <code>[bool](#bool)</code> | If True, models with a non-zero mean are considered. Defaults to True. | <code>True</code>
`blambda` | <code>[float](#float)</code> | Box-Cox transformation parameter.
If lambda="auto", then a transformation is automatically
selected using BoxCox.lambda.
The transformation is ignored if None.
Otherwise, data transformed before model is estimated. Defaults to None. | <code>None</code>
`biasadj` | <code>[bool](#bool)</code> | Use adjusted back-transformed mean for Box-Cox transformations.
If transformed data is used to produce forecasts and fitted values,
a regular back transformation will result in median forecasts.
If biasadj is True, an adjustment will be made to produce
mean forecasts and fitted values. Defaults to False. | <code>False</code>
`period` | <code>[int](#int)</code> | Number of observations per unit of time.
For example 24 for Hourly data. Defaults to 1. | <code>1</code>
